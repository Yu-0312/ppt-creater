# 可直接貼上的 prompt 模板（6 段）

這 6 段在任何 AI 都一字不改即可用。前 3 段做內容與規格，第 4 段做圖表，第 5、6 段收尾。

---

## 開口前先想清楚三件事（100 字以內）

把「幫我做簡報」改成說清楚：**現在的狀態 + 要給誰看 + 有什麼問題**。AI 在 Step 1 清點時能做出更精準的判斷，備援區的去留錯誤率直接減半。

格式：
> 這是 [文件類型＋背景]，要給 [受眾＋場合]，[限制或已知問題點]。

❌「幫我做成簡報」（AI 只能靠猜）

✅「這是 Q2 業績回顧的財報，下週要給董事會看，10 頁以內，CFO 說毛利率那個數字可能算錯了，圖表要重算。」

✅「這是上週三場跨部門會議的逐字稿混在一起，要整理成週一的管理層匯報，候選人相關資料不能出現在簡報上。」

不需要寫成完整段落，自然語言、口語都可以。**說清楚你現在在哪裡、問題在哪裡，比說「你是一個簡報專家請幫我」有效 10 倍。**

### 進階做法：把自己當設計師，讓 AI 找你的問題

與其請 AI 替你做簡報，更有效的做法是：**你先整理骨架，AI 分析你漏了什麼**。

不要外包大腦給 AI。你負責判斷「這份簡報要說什麼」，AI 負責告訴你你的判斷有沒有漏洞、邏輯有沒有跳躍。這個過程出來的簡報才是你真正想說的，不是 AI 猜出來的。

做法：
1. 先不要把文件丟給 AI。自己整理出一個大綱（哪怕只有 5 個重點、很粗也沒關係）
2. 把大綱 + 原始文件一起給 AI，用這個 prompt：

```
這是我打算在簡報裡說的大綱：
[你的大綱]

原始文件在附件。請告訴我：
1. 我的大綱漏了哪些原文裡的重要內容？
2. 這份簡報試圖表達的核心訊息是什麼？我有沒有說清楚？
3. 哪裡邏輯跳躍、哪裡聽眾可能看不懂？
不要幫我改，先只給我分析。
```

確認 AI 的診斷之後，再帶著修正過的大綱進入 Step 1 的清點流程。

---

## 第 1 段　提煉＋清點（整套最重要）

```
你是簡報內容設計師。把以下文件提煉成簡報稿，規則：
1. 先回報文件範圍：有幾個主題？哪些適合上簡報？範圍有疑義先問我，不要自己決定
2. 內容清點：原文每個區塊都必須標記去向——「上簡報」「備援區」或「另案處理＋建議格式」，不允許任何內容無聲消失
3. 敏感資料（候選人姓名／年齡／薪資／競業條款／個資／法務）一律攔進備援區，不直接寫進簡報稿
4. 上簡報的部分只保留對聽眾最有價值的 20%
5. 每頁最多 4 個要點，每點不超過 15 字
6. 總共 8-12 頁，最後一頁是結論或行動呼籲
先執行第 1、2、3 條，等我確認後再繼續。
```

---

## 第 2 段　風格規格（把「有設計感」翻成規格）

```
為這份簡報定義視覺規格，要求：
1. 用一句話定義氣質（從簡報內容本身找視覺語言，例如遊戲會議就用遊戲的美術設定）
2. 配色給色碼不給形容詞：背景、主文字、強調色、點綴色，每頁強調色不超過 2 處
3. 標題用襯線字、內文用無襯線字
4. 每種內容指定版型：數據→大數字卡片、預算→比例條、時程→時間軸，不准用表格
5. 封面和結語用深色滿版，前後呼應
```

---

## 第 3 段　生成

```
根據以上簡報稿與風格規格，生成簡報。交付要求：
1. 單一 HTML 檔案，零外部依賴，離線能開
2. 方向鍵和點擊翻頁、頂部進度條、右下角頁碼
3. 嚴格遵守風格規格，不准自由發揮
4. 內容只能來自簡報稿，不准自己加料
```

---

## 第 4 段　圖表鐵律（有數據時才用）

```
簡報需要圖表。鐵律：
1. 所有圖表必須用原始數據實際繪製與計算，禁止示意圖、禁止憑印象畫趨勢
2. 把原始數據完整嵌入檔案，圖由程式即時畫出
3. 畫圖前先重新計算文件聲稱的所有統計值（平均、斜率、R² 等），與原文比對，並展示計算過程
4. 算出來不一致的，標註在圖上，不准擅自改數字、不准捏造誤差、不准指控原作者
```

---

## 第 5 段　視覺驗收

```
預期：每頁強調色不超過 2 處。
實際：第 X 頁有 N 處強調色，視覺很吵。
請只修改第 X 頁，其他頁不准動。
```

### 說不清楚問題時：用類比逼出方向

如果看到成品「覺得不對但說不清楚哪裡不對」，不要繼續描述感覺——用日常生活的東西類比，讓 AI 從比喻裡提取方向：

```
這一版的感覺像是 [你不想要的東西]，我要的是 [你想要的東西]。
先只改封面頁，確認方向對了再全套改。
```

範例：

> 「這一版像是公司內網公告，我要的是 TED 演講的感覺。先改封面頁。」
>
> 「這一版太像傳統企業年報，我要的是科技新創 pitch deck 的張力。先改第 1 頁試試。」
>
> 「配色感覺像是學校講義，我要的是高端諮詢公司的冷靜感。只改配色，版型不要動。」

改完的新版本，再回到「預期 vs 實際」格式繼續縮小差距。**一次只改一頁確認方向，比一次說清楚所有問題再全改，成功率高很多。**

---

## 第 6 段　複盤封裝

```
把這次從文件到簡報的全部流程整理成 SOP：
每一步的 prompt 模板＋我做過的人工判斷＋這次學到的新規則。
```
